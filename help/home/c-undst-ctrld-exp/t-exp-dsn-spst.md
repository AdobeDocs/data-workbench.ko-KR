# 실험 설계 스프레드시트{#experiment-design-spreadsheet}

이 파일은 워크시트의 기능뿐만 아니라 실험에 대한 의사 결정의 기록으로도 작동합니다.

실험 디자인에 도움이 필요한 경우 Adobe에서 제공하는 실험 디자인 스프레드시트(기본적으로 VS Controlled Experiment Design.xls)를 사용할 수 있습니다.

실험 디자인 스프레드시트는 문제의 지표가 일부 기준을 충족하는 방문자의 백분율로 정의된 경우에만 유용한 통계 참조를 제공할 수 있습니다. 즉, 방문자 기반 지표 가설을 테스트할 때만 유용합니다.

**실험 디자인 파일을 사용하여 실험을 디자인하려면**

1. 웹 또는 응용 프로그램 서버에 대한 관리자 액세스 권한이 있는 경우 웹 클러스터의 [!DNL Sensor] 컴퓨터에서 [!DNL Sensor] 설치 폴더로 이동합니다. 관리자 액세스 권한이 없는 경우 Adobe 계정 관리자에게 문의하여 파일을 요청하십시오.
1. VS 제어 실험 Design.xls 파일을 엽니다. 원할 경우 이 파일의 이름을 변경할 수 있습니다.

   다음 페이지의 스프레드시트는 이 안내서 전체에서 사용되는 예제 가설을 테스트하기 위해 준비할 때 스프레드시트를 완료하는 방법의 예입니다.

   ![](assets/experimentdesigntop.png)

   ![](assets/experimentdesignmiddle.png)

   ![](assets/experimentdesignbottom.png)

나머지 필드는 입력한 값을 기반으로 계산되며 다음 표에 설명되어 있습니다.

| 필드 | 설명 |
|---|---|
| 애플리케이션 선택에 대한 예상 지표 | 웹 사이트에 필요한 지표 값에 대한 제목입니다. |
| 기간당 예상 방문자 수 | 이 필드는 일반적으로 스프레드시트에 의해 자동으로 계산됩니다. 웹 사이트에서 돌아온 방문자보다 많은 새 방문자를 받는 대부분의 요일에 의존합니다. 이 경우가 아닌 경우 이 셀의 계산을 실험 동안 예상된 실제 방문자 수로 덮어써야 합니다. |
| 유형 I 오류에 대한 계산된 Z 점수 | 잘못된 긍정 결과의 Z 점수. 이것은 중간 통계적 계산이다. |
| 유형 II 오류에 대한 계산된 Z 점수 | 잘못된 음수 결과의 Z 점수. 이것은 중간 통계적 계산이다. |
| 필요한 최소 방문자 수(Test+제어 그룹) | 지정된 신뢰 수준, 전원 수준 및 Z 점수를 충족하기 위해 실험에 필요한 최소 방문자 수는 [기간당 예상 방문자 수] 필드에 값의 백분율로 표시됩니다. |
| 필요한 최소 방문자 수(Test+제어 그룹) | 지정된 신뢰 수준, 전원 수준 및 Z 점수를 충족하기 위해 테스트에 필요한 최소 방문자 수입니다. 이 값은 방문자 섹션의 합계(보통 100%) 필드의 값보다 작거나 같아야 합니다. |
| 최소 실험 시간(일) | 지정된 신뢰 수준, 전원 수준 및 Z 점수를 충족하기 위해 실험을 실행하는 데 필요한 최소 일 수입니다. 이 계산된 수는 기간당 예상 방문자 수 필드에 설명된 것과 동일한 문제를 따릅니다. 재방문자가 많은 웹 사이트의 경우 최소 실험 시간(일) 필드는 최소 방문자 필요 필드의 값과 같은 고유 방문자 수를 보는 데 걸리는 예상 일수입니다. |
| 방문자 수 | 방문자 값에 대한 제목. |
| 테스트 그룹 | 테스트 그룹에 필요한 방문자 수입니다. |
| 제어 그룹 | 제어 그룹에 필요한 방문자 수입니다. |
| 합계(보통 100%) | 실험에 필요한 총 방문자 수. 이 값은 필요한 최소 방문자 수(Test+제어 그룹) 필드의 값과 같거나 커야 합니다. |
| 그룹 정확도 테스트(Target 신뢰 수준) | 테스트 그룹에 대해 계산된 지표의 측정된 값이 실제 값의 이 퍼센트 내에 있는 지정된 신뢰 수준과 같은 가능성이 있음을 나타내는 백분율입니다. |
| 그룹 정확도 제어(Target 신뢰 수준) | 제어 그룹에 대해 계산된 지표의 측정된 값이 실제 값의 이 백분율 내에 있을 수 있는 지정된 신뢰 수준과 같은 가능성이 있음을 나타내는 백분율입니다. |
| Z 점수(Target 정확도) | 주어진 값이 테스트 평균에서 나온 표준 편차의 수입니다. |
| 실제 신뢰 수준(Target 간격) | 실험에 대한 신뢰 수준. 신뢰도는 명시된 기대가 참이라는 확률을 측정합니다. |
| 실제 간격(Target 신뢰 수준) | 알 수 없는 모집단 매개 변수를 포함할 수 있는 값의 예상 범위를 제공하는 실험에 대해 달성된 신뢰 구간. 이 범위는 주어진 샘플 데이터 세트에서 계산됩니다. |

필요한 최소 방문자 수(테스트+제어 그룹) 필드에서 값을 확인해야 합니다...

![](assets/Experiment_Design_Min_Visitors.png)

및 [!DNL Visitors] 열의 [합계] 필드의 값과 비교할 수 있습니다.

![](assets/Experiment_Design_Total_Visitors.png)

실험 결과를 통계적으로 유효화하려면 합계(보통 100%) 필드의 값이 필요한 최소 방문자 수(Test+제어 그룹) 필드의 값과 같거나 커야 합니다.

제공된 입력과 함께 예제 워크시트는 95% 신뢰 비율(이 숫자를 늘릴 수 있지만 모든 제어 실험에 대해 제안된 최소 신뢰도임)을 충족하기 위해 10,475명의 방문자가 이 실험에 참여해야 한다는 것을 보여줍니다. 현재 설계된 실험에는 30,000명의 방문자가 포함되며, 이는 필요한 최소 방문자 수를 훨씬 넘는 수치입니다.

일 수를 동일하게 유지하는 경우 총 방문자 수가 회의를 계속하거나 필요한 최소 기준을 초과하는 한 신뢰 수준을 높일 수 있습니다.

1. 기록을 위해 파일을 저장한 다음 파일의 정보를 사용하여 실험 구성 스프레드시트를 사용하여 실험 결과를 구성합니다. 이 스프레드시트에 대한 자세한 내용은 [실험 구성 및 배포](../../home/c-undst-ctrld-exp/t-crt-ctrld-exp/c-cnfg-dply-exp.md#concept-50f1de0242904698937bb72b3ea1b429)를 참조하십시오.
