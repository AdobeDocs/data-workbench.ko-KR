## 업그레이드 요구 사항 및 Recommendations {#section-8704a9ac358246cd81233dd0982d534f}

Data Workbench에 대한 새 프로파일은 **소프트웨어 및 문서** 프로필에 있습니다.

```
Profiles - Current\DataWorkBench\ 
  English Translated\DataWorkBench_6.31-en-us\
```

![](assets/upgrade_3_21_profiles.png)

**업그레이드 서버:**

>[!NOTE]
>
>패키지에 제공된 기본 파일보다 우선하는 사용자 정의 프로파일을 [!DNL Base] 보유하고 있는 경우 이러한 사용자 정의 파일을 업데이트해야 합니다.

* **Meta.cfg 파일** 을 업데이트 [!DNL E:\..\Profiles\<your custom profile>\Context\meta.cfg)](FSU 서버)하여 업데이트된 암호 암호화를 설정하고, 이름 값 쌍 변형에 대한 항목을 추가하여 [쿼리 문자열 그룹화](../../../home/c-get-started/c-admin-intrf/c-query-que/c-query-string-grouping.md)기능을 활용합니다.

   1. FSU에서 [!DNL meta.cfg] 파일을 엽니다.
   1. 워크스테이션 구성 **[!UICONTROL Proxy Password]** 섹션의 &quot; [!DNL string"] 에서 &quot; [!DNL EncryptedString]&quot;로 데이터 유형 *을* 변경합니다.

      ```
      Proxy User Name = string: 
      Proxy Password = EncryptedString:   (from Proxy Password = String) 
      Use Address File = bool: true
      ```

   1. 새 항목을 추가하여 새 이름 값 쌍 변형을 활성화합니다. *BuildNameValuePair* 및 *ExtractNameValuePair*.

      작업 영역을 열고 관리 > **프로필 관리자를** 마우스 오른쪽 단추로 **클릭합니다**.

      컨텍스트 **에서****기본** 열에서 **meta.cfg** 파일을 **클릭한 다음 로컬**&#x200B;만들기를클릭합니다. 사용자 테이블 열에서 마우스 오른쪽 단추를 클릭하고 **열기** > 워크스테이션 **에서 선택합니다**.

      ![](assets/meta_cfg.png)

      * 새 창에서 **메타데이터를** 클릭하고 허용되는 하위 템플릿을 추가합니다.

      ![](assets/meta_cfg_child.png)

      * 변환 **을** 열고 새 템플릿을 추가합니다.

      ![](assets/meta_cfg_template.png)



* **향상된 빠른 병합**&#x200B;업데이트 변환 중 Data Workbench의 속도 향상을 활용하려면 매개 변수를 추가하거나 다음 구성 파일에 값을 변경합니다.

   * **Communications.cfg** ([!DNL E:\Server\Components\Communications.cfg])

      ```
      18 = SourceListServer:  
          URI = string: /SourceListServer/ 
          Listing Interval = int: 10 (new)
      ```

   * **Disk Files.cfg** (at [!DNL E:\Server\Components] and [!DNL E:\Server\Components for Processing Servers])

      ```
      Disk Cache Size (MB) = double: 1024 (from double: 256) 
      Disk Cache Read Limit (MB) = double: 768 (new)
      ```

   * **Log Processing Mode.cfg** ( [!DNL E:\Server\Profiles\<your profile>\Dataset\Log Processing Mode.cfg])

      ```
      <i>(changed)</i> 
      Batch Bytes = int: 268435456 
      Cloud Bytes = int: 268435456 
      Real Time FIFO Bytes = int: 268435456
      ```

      ```
      (new) 
      Cache Bytes = int: 32000000 
      Fast Input Decision Ratio = double: 200 
      Fast Input FIFO Bytes = int: 268435456 
      FIFO Hash Mask = int: 16383 
      Fast Merge Buffer Bytes = int: 536870912 
      Slow Merge Buffer Bytes = int: 268435456 
      Fast Merge Fan In = int: 64 
      Key Cache Size Logarithm = int: 21 
      Max Seeks = int: 512 
      Output Old Buffer Bytes = int: 536870912 
      Overflow FIFO Bytes = int: 67108864 
      Paused = bool: false
      ```

      >[!NOTE]
      >
      >* 빠른 병합 기능을 활용하려면 DPU당 최소 8GB의 RAM을 보유해야 합니다.
      >* 새로운 로그 처리 모드 매개 변수가 적용되는 경우 데이터 세트 간의 지속 시간인 저장 간격을 줄여야 할 수 있습니다. 변환이 보다 빠르게 완료되므로 데이터 세트 상태 파일에 이전 버전과 비교하여 간격 저장 간에 더 많은 데이터가 포함됩니다.

      >
      >  **[저장 간격] 설정을 1800초(30분)로 줄이는 것이 좋습니다.**&#x200B;기본 저장 간격은 3600초(1시간)입니다. (이 줄을 에 추가해야 하는 경우 [!DNL Log Processing Mode.cfg]앞에 두 개의 공백이 있습니다.)
      >
      >  
      ```
      >      Save Interval (sec) = int: 1800
      >  ```
      >
      >  * 새 로그 처리 모드 매개 변수는 모두 작업을 사용하여 **여러 행** 으로 구성된 데이터 세트에 적용되지 않는 **것이** 좋습니다. 이 설정을 사용하면 이 작업의 메모리 사용이 환경에 너무 편리할 수 있습니다.



* **DWB 통합 업데이트가 포함된 Adobe Target**. 새 내보내기 파일 [!DNL ExportIntegration.exe]은 Insight 서버()의 기존 [!DNL TnTSend.exe] 파일을 [!DNL E:\Server\Scripts\TnTSend.exe]대체합니다. 이 새로운 내보내기 파일은 Adobe Target 통합 및 새로운 기본 MMP(Marketing Profile) 및 Adobe Audience Manager과의 조정을 모두 지원합니다.

   Adobe Target 내보내기에 대해 다음 명령을 업데이트해야 합니다.

   `Command = string: TnTSend.exe`

   에서

   ```
   <filepath>
   Command = string: ExportIntegration.exe 
   </filepath>
   ```

   >[!NOTE]
   >
   >이는 버전 6.3 이전에 생성된 내보내기에만 영향을 줍니다.
   >
   >
   >이전 내보내기 프로세스를 사용하려면 다음을 시도해 볼 수도 있습니다.

   >    * 워크스테이션에서 새 테스트 및 Target 내보내기를 만듭니다.
   >* 서버/프로필/내보내기에 있는 이전 테스트 및 Target 내보내기를`<your profile>`수정합니다.


* **Adobe SC 프로필을 업데이트합니다.** 파일을 [!DNL Exclude Hit.cfg] 변경하려면 관련 [!DNL Decoding Instructions.cfg] 파일에서 필드를 선언해야 합니다.

   >[!NOTE]
   Adobe SC 프로필에 사용자 정의된 [!DNL Decoding Instructions.cfg] 파일이 포함되어 있는 경우 사용자 정의된 파일에 [!DNL DelimitedDecoder] 매개 변수를 포함해야 합니다.

   ```
   0 = DelimitedDecoder: 
      Delimiter = string: \t 
      Fields = vector: x items 
      …  
         5 = string: 
   Changed to: 
   
   5 = string: x-hit_source
   ```

   이 필드를 추가하면 [!DNL DelimitedDecoder] 기능 업데이트를 활용하고 이러한 업데이트로 인해 발생할 수 있는 로그 처리 문제를 방지할 수 있습니다.

**업그레이드 클라이언트:**

* **서버에서 클라이언트를 업데이트합니다**.

   서버가 업데이트되면 파일이 올바르게 구성된 경우 클라이언트가 자동으로 업데이트할 수 [!DNL Insight.cfg] 있습니다.

   1. Edit the [!DNL Insight.cfg] file.

      ```
      Update Software = bool: true
      ```

      Then **[!UICONTROL Save]**.

   1. 클라이언트를 종료하고 시작합니다.
   1. 프로필에 연결합니다.

      클라이언트가 Data Workbench 6.3으로 자동 업그레이드됩니다.

   1. 클라이언트를 종료합니다.
   1. 템플릿 [!DNL Insight.cfg]

      * 변경 [!DNL Proxy Password = string:]

         에서 [!DNL Proxy Password = EncryptedString:]

         이전 프록시 주소 및 프록시 암호 값을 제거합니다.

      * **[!UICONTROL Save]**&#x200B;를 참조하십시오.
   1. 클라이언트를 시작합니다.
   1. 템플릿 [!DNL Insight.cfg].

      * 모든 서버의 프록시 암호를 입력하고 저장을 입력합니다.
      * 모든 서버의 프록시 주소를 입력하고 저장을 입력합니다.

         >[!IMPORTANT]
         클라이언트 내에서 프록시 주소와 프록시 암호를 입력하고 저장해야 합니다.
   1. 프로필에 연결합니다.
   >[!NOTE]
   * 계정 잠금이 발생하지 않도록 하려면 정확한 업그레이드 순서에 따릅니다. 계정이 잠긴 경우, 나열된 정확한 시퀀스에 필요한 모든 변경 사항을 수행하고 작업을 저장하고 클라이언트를 종료하십시오. 잠금이 해제될 때까지(약 45분) 기다렸다가 클라이언트를 다시 시작합니다.
   * 암호는 Windows Credential Vault에 저장되기 때문에 클라이언트에서 수정해야 합니다.


* **권장 사항**:**새로운 Windows Aero 테마. ** Windows Aero 테마를 사용하여 클라이언트 애플리케이션의 모양을 업그레이드하십시오.

   *심선은* 중국어와 일본어로 사용될 수 있다. 일본어로 2바이트 문자를 쓰려고 하는 경우 *MS Mincho도 포함해야 합니다*. 에서 이러한 글꼴을 활성화하려면 이러한 매개 변수 [!DNL Insight.cfg]를 추가할 수 있습니다.

   ```
   0 = string: Arial 
   1 = string: SimSun 
   2 = string: MS Mincho
   ```

   이러한 글꼴은 워크스테이션 구성 파일에 나열되어야 합니다. [!DNL Insight.cfg.]

**Adobe Analytics Premium으로 업그레이드**

Data Workbench에서 **최적 속성을** 실행하려면, Adobe Analytics Premium을 지원하기 위해 클라이언트, 서버 및 보고서 서버(.pem 파일)에 대한 Adobe ClientCare로부터 새 인증서를 받아야 합니다. 새 인증서 각각에 이 매개 변수가 있습니다.

Product = Premium
```

Navigate to `Profiles - Current\DataWorkBench\<language>\Data WorkBench_6.30-en-us\Premium_6.30_en-us.zip`.  서버에서 **프리미엄** 프로필을 로드하면 *프리미엄* 매개 변수를 사용자 지정 [!DNL Profile.cfg] 파일에 추가해야합니다. 이를 통해 사용자 정의 프로필에는 메뉴, 시각화 및 작업 영역이 Adobe Analytics Premium의 일부로 포함됩니다.

## 수정된 버그 {#section-e1e3df3da3594987a325344cd5e8b855}

* 밀도 맵 **시각화에** 가장 큰 요소가 누락되는 문제가 해결되었습니다.
* 요소 **영역이 지표** 값의 비율을 나타내지 않는 밀도 맵의 문제가 해결되었습니다.
* 지표를 **[!UICONTROL Finders]** 패널에서 지표 열 외부에 있는 지표 범례로 드래그하면 작업 영역에서 범례가 삭제되는 문제가 해결되었습니다.
* 인쇄 페이지 **[!UICONTROL Print Workspace]** 에 저작권 정보 **[!UICONTROL Sidebar]** 를 **[!UICONTROL Both]** 사용하고옵션을 사용하지 않는 문제가 해결되었습니다.

## 알려진 문제 {#section-751e8698e9f14b75a85f63253782be41}

* AMD **Radeon™ 그래픽 카드** 사용자는 최신 그래픽 드라이버로 업데이트해야 합니다. 일부 초기 버전의 드라이버에서는 openGL 3.2를 지원하지만 일치하지 않는다고 주장합니다.
* 헤더 선언 없이 **세그먼트 내보내기** 구성으로 생성된 출력이 파일 시작 부분에 허위 헤더가 나타나고 첫 번째 행 세트와 충돌하는 결과가 발생할 수 있습니다.
* **Dimension 추가에** 확장 Dimension만 *표시됩니다*. 파인더 **** 도구를 사용하여 차원을 테이블로 드래그하는 것이 해결 방법입니다.

* 3D 산포도 시각화에 콜아웃이 포함되어 있으면 확대/축소에 시각화의 테두리 밖의 플롯이 표시될 수 있습니다. 이 문제를 해결하려면 먼저 3D 분포도를 확대/축소한 다음 시각화에 콜아웃을 추가하십시오.
* 작업 영역 이름을 바꾸면 원격 데스크톱 세션에서 워크스테이션 사용이 중단됩니다.
* 목록 항목
* **기존 세그먼트 내보내기 파일은 출력 형식 필드에 인용 부호가 포함되어 있지 않더라도 큰 따옴표로 출력됩니다** .

   *해결 방법*:이 세 줄을 .export 파일에 추가합니다. 이러한 값을 설정해도 MMP 통합이 트리거되지 않고(다른 구성 필드가 필요함) 원하지 않는 자동 이스케이프 처리를 무시할 수 있습니다.

   ```
     MMP Configuration = MMPConfiguration: 
       MMP Segment Name = string: UNESCAPE DUMMY 
       MMP Visitor ID Field = string: [Specify a Dimension from the output of 
                                       the current export]
   ```

   (첫 번째 줄에는 두 개의 앞에 공백이 있고 다음 줄은 네 개(4)입니다. 현재 내보내기의 출력에 있는 Dimension은 에서 참조되어야 [!DNL MMP Visitor ID Field]합니다.)
