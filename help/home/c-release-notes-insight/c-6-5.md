## 업그레이드 요구 사항 {#section-84f8f38e9c424d7788c79146fce5a6fe}

Data Workbench 6.5로 업그레이드할 때 이러한 요구 사항과 권장 사항을 따르십시오.

* **`Components for Processing Servers\Communications.cfg`** 파일을 변경하려면 DWB 6.5 릴리스에 대해 이 파일을 업데이트해야 합니다. *SourceListServer*, *SegmentExportServer* 및 *NormalizeServer* 항목이 제거되었습니다. (DPU에서 *sourcelist*, *세그먼트 내보내기* 또는 *서버 정규화*&#x200B;를 실행해서는 안 됩니다.)

## 시스템 업데이트 {#section-bcdd732af53249b893d30bb42eeb14fc}

이번 릴리스에서 이러한 기능의 이름이 바뀌었거나 삭제되었거나, 설치 파일이나 폴더가 재구성되었습니다.

* 상관 관계 Chord, 상관 관계 매트릭스, 연결 Chord, 연결 매트릭스, 성향 점수 및 최적 속성 시각화는 이제 **다중 전달 시각화**&#x200B;가 되어 보고서 서버에서 실행할 수 있습니다.

   >[!IMPORTANT]
   >
   >작업 공간에는 하나의 다중 전달 시각화만 있을 수 있습니다.
   >
   >작업 공간에 다중 전달 시각화가 두 개 이상 있는 경우 보고서 서버가 기본적으로 보고서를 생성하지 못하고 다음 오류가 발생합니다.
   >
   >
   ```
   >Too many Multipass visualizations in workspace ....... (has #, 1 allowed).
   >```
   >
   >[!DNL ReportServer.cfg] 파일을 업데이트하거나 *보고* 섹션에서 이 줄을 기존 파일에 추가하여 이 오류를 방지합니다.   >
   >
   >
   ```
   >Max Multipass Per Slice = int: 1
   >```
   >
   >*슬라이스당 최대 다중 전달* 값은 1로 설정되어야 합니다(*알려진 문제* 참조).

* [!DNL Server\Menu\Admin] 폴더의 파일을 변경했습니다.

   * 조회가 제거되었습니다.
   * 아카이브 파일 및 사용자 파일을 추가했습니다.
   * 수정된 파일, 내보내기 파일, 로그 파일.

   업데이트된 파일은 다음과 같습니다.

   ```

## 알려진 문제 {#section-350ae121ca3442babbef4d0a2b027df1}

* *Windows 인증서 저장소*&#x200B;는 영어로만 제한됩니다(중국어 또는 일본어 지원 없음).
